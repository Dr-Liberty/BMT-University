1. Executive Summary

The Vision

Transform BMT University into a white-label SaaS platform where any crypto project can launch their own branded learn-to-earn university and pay students in their native token. V1 will be EVM-compatible, with future expansion to all major chains.

2. Technical Architecture (V1 - EVM Only)

Multi-Tenant Design

Component Current (BMT) Crypto University
Database Single tenant tenant_id on all tables
Branding Hardcoded JSON config per tenant
Payouts $BMT only Tenant-configured ERC-20
Domains bmtuniversity.com Custom domains per tenant

Chain Abstraction Layer

  • V1: EVMAdapter class handling all EVM chains (Ethereum, Polygon, Base, Kasplex)
  • V2+: Add SolanaAdapter, CosmosAdapter via plugin architecture
  • Unified ChainAdapter interface for payouts, balance checks, and token info

Shared vs Isolated Components

  • Shared: Auth system, course engine, analytics core, security/rate limiting
  • Per-Tenant: Branding assets, payout wallets, course catalogs, student data

3. Business Model & Monetization

Pricing Tiers

Tier Price Features Payout Fee
Starter Free 1 course, 500 students/mo, Crypto U branding 5%
Growth $299/mo Unlimited courses, custom domain, referrals, analytics 2%
Enterprise Custom SLAs, dedicated support, API access, on-chain automation 0.5-1%

Revenue Streams

  1. Subscriptions (ARR) - Monthly/annual SaaS fees
  2. Transaction Fees - Percentage of every token payout
  3. Premium Features - Featured marketplace placement, NFT certificates, API access
  4. Services - Token issuance consulting, liquidity bootstrapping

4. MVP Feature Set (V1)

Must Have (Launch)

  • Tenant onboarding wizard (connect wallet, configure token, upload branding)
  • Course builder with modular content/quiz engine
  • Student marketplace (browse all universities)
  • Wallet auth + tenant-specific ERC-20 payouts
  • Analytics dashboard (enrollments, completions, rewards)
  • Referral tracking per tenant
  • Admin panel for tenant managers
  • Stripe integration for SaaS billing

Defer to V2+

  • Multi-chain adapters (Solana, Cosmos, Move)
  • NFT credentials/certificates
  • Advanced gamification (leaderboards, achievements)
  • DAO governance features
  • Discord/Telegram integrations

5. Go-To-Market Strategy

Target Customers

  1. L2 Ecosystems - Base, Arbitrum, Optimism, Kasplex
  2. DeFi Protocols - Uniswap, Aave, Compound
  3. Gaming/NFT Projects - Need to explain complex mechanics
  4. DAO Accelerators - Graduate programs, contributor training
  5. Token Projects - Use education as token distribution

Sales Approach

  • Founder-led sales - You're the first customer (BMT)
  • Case study marketing - Document BMT University metrics publicly
  • Ecosystem partnerships - Partner with chain foundations for grants
  • Developer relations - API/SDK docs, tutorials, bounties
